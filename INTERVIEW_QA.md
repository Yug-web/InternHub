## Section 1: MVC Architecture (Q1-Q5)

---

### Q1. What is MVC? Explain how InternHub uses it.

MVC stands for Model-View-Controller — a design pattern that separates an application into three components:

- **Model** = Data shape. Examples: `Company.cs`, `Application.cs` — C# classes mapping to DB tables
- **View** = Displays data. The `.cshtml` Razor files in `/Views/`
- **Controller** = Handles requests, calls DB, returns Views. Examples: `CompaniesController.cs`, `ApplicationsController.cs`

**Flow in InternHub:**
```
Browser → GET /Applications
       → ApplicationsController.Index()
       → Queries DB using EF Core + LINQ
       → Returns View(applications)
       → Razor renders HTML table
       → Browser displays it
```

---

### Q2. What is a Controller? How did you create one?

A Controller is a C# class that:
1. Inherits from `Controller` (base class in ASP.NET Core MVC)
2. Has action methods corresponding to HTTP routes
3. Returns `IActionResult` — a View, Redirect, or JSON

In InternHub, `ApplicationsController` has:
```csharp
public async Task<IActionResult> Index(string? search)   // GET /Applications
public async Task<IActionResult> Create()                // GET /Applications/Create
[HttpPost] public async Task<IActionResult> Create(...)  // POST /Applications/Create
```

URL convention: `{controller}/{action}/{id?}` maps automatically.

---

### Q3. What is a ViewModel? Why did you use `DashboardViewModel`?

A ViewModel is a custom class made specifically for a View. It holds only the data the View needs, combining data from multiple models.

The Dashboard needed both numbers (TotalApplications, SelectedCount) AND lists (RecentApplications, UpcomingDeadlines). No single model had all this, so I created:

```csharp
public class DashboardViewModel {
    public int TotalApplications { get; set; }
    public int SelectedCount { get; set; }
    public List<Application> RecentApplications { get; set; } = new();
    public List<Deadline> UpcomingDeadlines { get; set; } = new();
}
```

The Controller builds it and passes it: `return View(viewModel)`.

### Q4. How does routing work in ASP.NET Core MVC?

Routing maps a URL to a Controller action. In `Program.cs`:
```csharp
app.MapControllerRoute(
    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");
```

URL → Controller → Action mapping:
- `/` → HomeController.Index
- `/Companies` → CompaniesController.Index
- `/Applications/Edit/3` → ApplicationsController.Edit(id=3)
- `/Deadlines/Create` → DeadlinesController.Create

### Q5. What is `[ValidateAntiForgeryToken]` and why use it?

It prevents CSRF (Cross-Site Request Forgery) attacks. When a form is submitted, ASP.NET Core validates a hidden token generated by `@Html.AntiForgeryToken()`. If an attacker's malicious website tries to submit a fake DELETE form, the token won't match and the request is rejected.

Every POST action in InternHub has it:
```csharp
[HttpPost]
[ValidateAntiForgeryToken]
public async Task<IActionResult> Create(...) { }
```

## Section 2: Entity Framework Core (Q6-Q10)

---

### Q6. What is Entity Framework Core? How did you use it?

EF Core is an ORM (Object-Relational Mapper) — it lets you work with the database using C# objects instead of raw SQL.

Instead of writing `SELECT * FROM Applications WHERE Status = 'Selected'`, I write:
```csharp
var count = await _context.Applications
    .CountAsync(a => a.Status == ApplicationStatus.Selected);
```

EF Core generates the SQL automatically.

---

### Q7. What is a DbContext? Explain your `AppDbContext`.

`DbContext` is the main EF Core class. It manages the DB connection, tracks changes, and executes operations.

```csharp
public class AppDbContext : DbContext {
    // Each DbSet maps to a database table
    public DbSet<Company> Companies { get; set; }
    public DbSet<Application> Applications { get; set; }
    public DbSet<Interview> Interviews { get; set; }
    public DbSet<Deadline> Deadlines { get; set; }
}
```

It's registered in `Program.cs`:
```csharp
builder.Services.AddDbContext<AppDbContext>(options =>
    options.UseSqlServer(connectionString));
```

And injected into controllers via Dependency Injection.

### Q8. What is Code-First approach? How is it different from Database-First?

- **Code-First**: Write C# models first → EF generates DB schema via migrations *(InternHub)*
- **Database-First**: Start with existing DB → EF generates C# models from tables

InternHub Code-First process:
1. Wrote `Company.cs`, `Application.cs`, etc.
2. Ran `dotnet ef migrations add InitialCreate`
3. EF generated SQL to create tables
4. Ran `dotnet ef database update` to apply

---

### Q9. What are EF Core Migrations?

Migrations version-control your database schema. Each migration is a C# file describing schema changes.

```csharp
protected override void Up(MigrationBuilder migrationBuilder)
{
    migrationBuilder.CreateTable("Companies", ...);
    migrationBuilder.CreateTable("Applications", ...); // with FK
}

protected override void Down(MigrationBuilder migrationBuilder)
{
    migrationBuilder.DropTable("Applications");
    migrationBuilder.DropTable("Companies");
}
```

`Up()` applies the migration. `Down()` reverses it (rollback).

### Q10. What is Eager Loading? When did you use `Include()`?

Eager Loading means loading related data in the same SQL query using a JOIN.

By default if you load an `Application`, the related `Company` is null. You need `.Include()`:

```csharp
// Dashboard — loads Company with each Application in one query
RecentApplications = await _context.Applications
    .Include(a => a.Company)              // JOINs Companies table
    .OrderByDescending(a => a.CreatedAt)
    .Take(5)
    .ToListAsync();
```

Without `Include()`, `app.Company.Name` throws a NullReferenceException.

## Section 3: LINQ (Q11-Q13)

---

### Q11. What is LINQ? Give 3 examples from InternHub.

LINQ (Language Integrated Query) lets you query collections or databases in C# syntax.

**Example 1 — CountAsync (aggregate):**
```csharp
SelectedCount = await _context.Applications
    .CountAsync(a => a.Status == ApplicationStatus.Selected);
```

**Example 2 — Where + OrderBy + Take:**
```csharp
UpcomingDeadlines = await _context.Deadlines
    .Where(d => d.DueDate >= DateTime.Now.Date)
    .OrderBy(d => d.DueDate)
    .Take(5)
    .ToListAsync();
```

**Example 3 — Search with Contains:**
```csharp
query = query.Where(c =>
    c.Name.ToLower().Contains(search) ||
    c.Industry.ToLower().Contains(search));
```

---

### Q12. What is the difference between `ToList()` and `ToListAsync()`?

- `ToList()` — blocks the thread until the DB query completes
- `ToListAsync()` — async, frees the thread while waiting for DB

In ASP.NET Core, `async/await` with `ToListAsync()` is best practice because web servers handle many requests simultaneously. Blocking threads wastes server capacity. All InternHub controller actions are `async Task<IActionResult>`.

---

### Q13. What is `AsQueryable()` and why did you use it?

`AsQueryable()` creates a deferred query — the SQL is not executed until `ToListAsync()` is called. This lets you conditionally add WHERE clauses:

```csharp
var query = _context.Applications.Include(a => a.Company).AsQueryable();

if (!string.IsNullOrEmpty(search))
    query = query.Where(a => a.Role.Contains(search));   // add search filter

if (!string.IsNullOrEmpty(statusFilter))
    query = query.Where(a => a.Status == parsedStatus);  // add status filter

var results = await query.ToListAsync(); // ONE SQL query with all filters
```

Without `AsQueryable()`, you'd load all rows from DB first, then filter in memory — very slow.

## Section 4: Database Relationships (Q14-Q16)

---

### Q14. What database relationships does InternHub have?

InternHub has One-to-Many relationships:
- One `Company` → Many `Applications`
- One `Company` → Many `Interviews`

Configured in `AppDbContext.OnModelCreating()`:
```csharp
modelBuilder.Entity<Application>()
    .HasOne(a => a.Company)
    .WithMany(c => c.Applications)
    .HasForeignKey(a => a.CompanyId)
    .OnDelete(DeleteBehavior.Cascade);
```

In the model:
```csharp
// Application.cs — FK column + navigation property
public int CompanyId { get; set; }
public Company? Company { get; set; }

// Company.cs — collection navigation property
public ICollection<Application> Applications { get; set; } = new List<Application>();
```

---

### Q15. What is Cascade Delete? Did you use it?

Cascade Delete means when a parent record is deleted, all related child records are automatically deleted.

In InternHub: deleting Google from Companies automatically deletes all Google's Applications and Interviews. I configured this with `OnDelete(DeleteBehavior.Cascade)` and warn the user on the Delete confirmation page.

---

### Q16. What is a Foreign Key? How does it work in InternHub?

A Foreign Key is a column in one table that references the Primary Key of another table. It ensures you can't have an Application for a Company that doesn't exist.

```csharp
public class Application {
    public int CompanyId { get; set; }    // FK — stores Company's Id
    public Company? Company { get; set; } // Navigation property — gives access to Company object
}
```

In the UI, the Create Application form shows a dropdown of Companies. The selected Company's `Id` is saved as `CompanyId` in the Applications table.

## Section 5: Validation, OOP & Best Practices (Q17-Q20)

---

### Q17. How did you implement validation in InternHub?

I used Data Annotations — attributes on model properties:

```csharp
public class Company {
    [Required(ErrorMessage = "Company Name is required.")]
    [StringLength(100, ErrorMessage = "Cannot exceed 100 characters.")]
    public string Name { get; set; }

    [EmailAddress(ErrorMessage = "Please enter a valid email.")]
    public string? HREmail { get; set; }

    [Url(ErrorMessage = "Please enter a valid URL.")]
    public string? Website { get; set; }
}
```

In the controller, I check `ModelState.IsValid`. If invalid, I re-show the form with error messages:
```csharp
if (ModelState.IsValid) {
    _context.Add(company);
    await _context.SaveChangesAsync();
    return RedirectToAction(nameof(Index));
}
return View(company); // Form re-displayed with validation errors
```

### Q19. What is Dependency Injection? How is it used in InternHub?

DI is a pattern where objects receive their dependencies from outside rather than creating them internally.

Without DI (tightly coupled — BAD):
```csharp
public class CompaniesController {
    private AppDbContext _context = new AppDbContext(); // creates its own instance
}
```

With DI (InternHub — GOOD):
```csharp
// Register once in Program.cs
builder.Services.AddDbContext<AppDbContext>(options => options.UseSqlServer(...));

// ASP.NET Core automatically injects it
public class CompaniesController : Controller {
    private readonly AppDbContext _context;
    public CompaniesController(AppDbContext context) {
        _context = context; // injected by the framework
    }
}
```

Benefits: loosely coupled, easy to test, single DB connection managed by the framework.
